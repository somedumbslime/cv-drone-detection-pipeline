# Архітектура

[English version](architecture.md)

## Огляд Pipeline

Цей репозиторій демонструє end-to-end workflow у computer vision для задачі детекції об'єктів (дрони):

1. Сирі відео/зображення зберігаються в `data/raw/`.
2. Кадри витягуються у `data/interim/`.
3. Майже дублікати видаляються.
4. Дані розмічаються у CVAT.
5. CVAT-експорт конвертується у YOLO-формат і ділиться на train/val/test.
6. YOLO-модель навчається, найкращі ваги зберігаються в `models/weights/`.
7. Навчені ваги експортуються у ONNX в `models/onnx/`.
8. Інференс доступний через скрипти (`src/inference/`) і FastAPI (`src/api/`).

## Потік Даних

```text
Сирі відео / зображення
        -> Витяг кадрів
        -> Дедуплікація
        -> Розмітка у CVAT
        -> Конвертація + split у YOLO
        -> Навчання YOLO
        -> Експорт у ONNX
        -> Інференс (зображення/відео)
        -> FastAPI endpoint /predict
```

## Автоматизовані Та Ручні Кроки

Автоматизовано:

- витяг кадрів (`src/data/extract_frames.py`)
- дедуплікація (`src/data/deduplicate.py`)
- конвертація датасету та split (`src/data/prepare_dataset.py`)
- навчання (`src/training/train_yolo.py`)
- оцінка (`src/training/evaluate.py`)
- експорт ONNX (`src/training/export_onnx.py`)
- скрипти інференсу та API

Ручні кроки:

- збір вихідних даних
- рішення щодо розмітки та QA у CVAT
- оновлення таксономії класів
- аналіз помилок моделі та ітерації

## Дизайн-Рішення

- Архітектура максимально проста й захищувана на співбесіді.
- Фокус на чітких етапах ML pipeline без зайвої інфраструктури.
- Використовується одна сім'я детекторів (YOLO), щоб концентруватися на якості даних і відтворюваності.
- API залишається мінімальним serving-шаром для прогнозів.
