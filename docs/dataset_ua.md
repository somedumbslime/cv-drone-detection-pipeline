# Датасет

[English version](dataset.md)

## Джерела Даних

Проєкт використовує візуальні дані, пов'язані з дронами (відео/зображення), які перетворюються у розмічені приклади для object detection.

## Етапи Датасету

- `data/raw/`: сирі медіа до обробки
- `data/interim/`: витягнуті й очищені кадри до фінального пакування
- `data/annotations/`: артефакти розмітки, експортовані з CVAT
- `data/processed/`: фінальні YOLO-ready split-и

## Workflow Розмітки

1. Витягнути кандидатні кадри.
2. Видалити майже дублікати.
3. Завантажити відібрані кадри в CVAT.
4. Розмітити bounding boxes.
5. Експортувати labels у YOLO-сумісному форматі.
6. Перетворити експорт у стандартну YOLO-структуру та split train/val/test.

## Класи

Назви класів визначаються в `configs/dataset.yaml` і синхронізуються скриптом `src/data/prepare_dataset.py` на основі метаданих CVAT-експорту.

## Стратегія Split

Коефіцієнти split налаштовуються в `configs/train_config.yaml` у секції `data_preparation.split`.

За замовчуванням:

- train: 70%
- val: 20%
- test: 10%

Split відтворюваний завдяки фіксованому random seed.

## Контроль Якості

- перевірка відповідності пар image-label під час підготовки датасету
- створення порожніх label-файлів тільки якщо файл розмітки відсутній
- перевірка цілісності YOLO-структури (`images/*`, `labels/*`)

## Нотатки Для Співбесіди

Будь готовий пояснити:

- чому дедуплікація покращує узагальнення
- навіщо потрібен split train/val/test
- як CVAT-експорт перетворюється у training-ready дані
- обмеження датасету та ризики дисбалансу класів
